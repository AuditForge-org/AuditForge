Scan Solidity in three steps

  1. Submit — paste Solidity source, enter a verified contract address, or link a GitHub repository.
  2. Scan — six engines run in parallel inside isolated, network-disabled sandboxes; a typical scan finishes in under 60 seconds.
  3. Review — get a 0–100 score, findings ranked by severity and engine consensus, an AI brief, and a shareable PDF.

What it catches

Audit Forge's engines cover the vulnerability classes that static and symbolic analysis detect well — reentrancy (SWC-107), access-control and tx.origin misuse, unchecked external calls, unsafe delegatecall, integer and rounding issues, weak randomness, timestamp dependence, and centralization risk. Every finding is mapped to its SWC category so you can look up the underlying weakness.

Why "free" doesn't mean "noisy"

Most free scanners flood you with single-tool false positives. Audit Forge weights each finding by how many independent engines agree and discounts lone, low-confidence and lint-only findings — so a clean, well-audited contract scores well and a genuinely risky one doesn't. It's free, open source (AGPL-3.0), and signed-in users get higher daily limits.

Automated scanning is a first pass, not a guarantee — always get a professional manual audit before deploying a contract that holds funds.
