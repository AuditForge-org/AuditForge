What a Web3 smart contract audit checks

Smart contracts are immutable once deployed, and Web3 protocols hold real value — so a single overlooked bug can be catastrophic. Audit Forge surfaces the classes of issues that automated analysis catches well:

  • Reentrancy — external calls that let an attacker re-enter before state is updated.
  • Access control — missing or incorrect ownership / permission checks on sensitive functions.
  • Unchecked low-level calls and unsafe value transfers.
  • Arithmetic and rounding issues, and unsafe use of delegatecall.
  • Timestamp and randomness dependence, and centralization / admin-key risk.

Why multi-engine consensus matters

Any single scanner produces false positives. Audit Forge runs six engines — Slither, Mythril, Aderyn, Semgrep, Solhint and Echidna — and weights each finding by how many independent tools agree. A finding confirmed by several engines is far more likely to be real than one raised by a single tool, which keeps the signal high and the noise low.
